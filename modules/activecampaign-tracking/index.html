ActiveCampaign Tracking Module

Overview​

ActiveCampaign supports several tracking features including site and event tracking.

Site Tracking is a powerful feature that connects your ActiveCampaign marketing and sales processes to activity on your website. It allows ActiveCampaign to track and log that website activity and then trigger and control actions in real-time as contacts visit your site and view specific pages.

Event Tracking allows you to collect data on a wide variety of contact behavior. You can create an event for many activities on your website and assign it a value and store that within ActiveCampaign. The captured event data can then be used to improve your marketing and sales processes as well as to trigger and control automations, create segment conditions, and personalise campaigns.

The ActiveMember360 ActiveCampaign Tracking module fully integrates ActiveCampaign site and event tracking with your WordPress website.

It allows any users to implement ActiveCampaign event tracking without the need for Javascript development expertise nor API and PHP skills.

Activation​

  1. In WordPress administrator main navigation click ActiveMember360, Modules.
  2. In ActiveMember360 Modules tabs click ActiveCampaign Tracking.
  3. Set Enable the ActiveCampaign Tracking module to Yes/On.
  4. Click Save Settings.

Site Tracking​

  1. In WordPress administrator main navigation click ActiveMember360, Modules.
  2. In ActiveMember360 Modules tabs click ActiveCampaign Tracking.
  3. Set Enable Site Tracking to Yes/On.
  4. Click Save Settings.

Once Site Tracking is enabled ActiveMember360 will include the ActiveCampaign site tracking code into your site for all WordPress URLs.

For logged in users the email address will also be set for the tracking code.

Important

For ActiveCampaign site tracking to function you must ensure that within your ActiveCampaign application under Settings, Tracking, Site Tracking is enabled irrespective of the settings in ActiveMember360.

Also the root domain/subdomain of your site must be whitelisted for site tracking within ActiveCampaign under Settings, Tracking, Whitelist and Install Code, Website URL and Include all website pages checked.

If the ActiveMember360 ActiveCampaign Tracking module is enabled for Site Tracking you should ensure that neither the ActiveCampaign WordPress plugin is enabled to install the ActiveCampaign site tracking code, nor any other service or plugin that does the same.

To include the site tracking code more than once will result in inaccurate tracking statistics.

When the ActiveMember360 ActiveCampaign Tracking module is enabled for Site Tracking it will automatically add the logged in users email address to the tracking code.

Event Tracking​

To enable/activate Event Tracking, follow the steps indicated below:

  1. In WordPress administrator main navigation click ActiveMember360, Modules.
  2. In ActiveMember360 Modules tabs click ActiveCampaign Tracking.
  3. Set Enable Event Tracking to Yes/On.
  4. Click Save Settings.
Note

The Site Tracking module does not need to be enabled to use the Event Tracking module.

Enabling Event Tracking in ActiveMember360 will automatically enable Event Tracking in ActiveCampaign under Settings, Tracking, Event Tracking.

Standard Events​

When Event Tracking tracking is enabled some predefined WordPress events can be configured for tracking.

  1. In WordPress administrator main navigation click ActiveMember360, Modules.
  2. In ActiveMember360 Modules tabs click ActiveCampaign Tracking.
  3. Ensure Enable Event Tracking is set to Yes/On and navigate to Standard Events section.
  4. In the Login Event input specify an event name of your choice to identify the login events of your contacts. This event name will have an event value showing the domain of the successful login.
  5. In the Logout Event input specify an event name of your choice to identify the logout events of your contacts. This event name will have an event value showing the domain of the successful logout.
  6. In the Registration Event input specify an event name of your choice to identify the new WordPress user registration event for your contacts. This event name will have an event value showing the domain of the successful user registration.
  7. Click Save Settings.
Note

Any event name unspecified, i.e. left blank, will be ignored and no event logging will take place for that event.

Your specified events will be automatically created within ActiveCampaign. However they will not appear under Settings, Tracking, Event Tracking, Event until they have been tracked and logged at least once.

For the Registration Event

  • If the new user is added to WordPress by ActiveMember360 during their first successful login process that will show as a Registration event. The Login event will also be logged if configured.

  • If you allow the user to register in WordPress using the native WordPress registration process that will show as a Registration event. Please note we do not recommend the use of this method.

  • If you are running WooCommerce and the user is created in WordPress after a successful purchase that will show as a Registration event.

WooCommerce Events​

When WooCommerce is installed and activated, and the ActiveMember360 WooCommerce Extension is also enabled, some predefined events based upon WooCommerce order status can be configured for tracking.

  1. In WordPress administrator main navigation click ActiveMember360, Modules.
  2. In ActiveMember360 Modules tabs click ActiveCampaign Tracking.
  3. Ensure Enable Event Tracking is set to Yes/On and navigate to WooCommerce Events section.
  4. In the WooCommerce Payment Completed Event input specify an event name of your choice to identify the event when a WooCommerce Payment is completed for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  5. In the WooCommerce Purchased Product Event input specify an event name of your choice to identify the event when a WooCommerce Product is purchased by your contacts. This event will be logged twice in ActiveCampaign, one event value showing the WooCommerce Product ID and the second event value showing the WooCommerce Product Name.
  6. In the WooCommerce Pending Payment Event input specify an event name of your choice to identify the event when a WooCommerce Pending Payment Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  7. In the WooCommerce Processing Event input specify an event name of your choice to to identify the event when a WooCommerce Processing Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  8. In the WooCommerce On Hold Event input specify an event name of your choice to to identify the event when a WooCommerce On Hold Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  9. In the WooCommerce Completed Event input specify an event name of your choice to to identify the event when a WooCommerce Completed Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  10. In the WooCommerce Cancelled Event input specify an event name of your choice to to identify the event when a WooCommerce Cancelled Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  11. In the WooCommerce Refunded Event input specify an event name of your choice to to identify the event when a WooCommerce Refunded Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  12. In the WooCommerce Failed Event input specify an event name of your choice to to identify the event when a WooCommerce Failed Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  13. In the WooCommerce Draft Event input specify an event name of your choice to to identify the event when a WooCommerce Draft Order Status is set for your contacts. This event name will have an event value showing the WooCommerce Order ID.
  14. Click Save Settings.
Note

Any event name unspecified, i.e. left blank, will be ignored and no event logging will take place for that event.

Your specified events will be automatically created within ActiveCampaign. However they will not appear under Settings, Tracking, Event Tracking, Event until they have been tracked and logged at least once.

DOM Events​

In addition to the Standard Events and WooCommerce Events supported by the ActiveMember360 ActiveCampaign Tracking module events can also be tracked and logged in ActiveCampaign for logged in users who exist as contacts within ActiveCampaign using onClick and onSubmit DOM (Document Object Model) events.

onClick​

To define/set an onClick logging event for any DOM HTML element that supports onClick use the HTML5 data attributes data-mbr-eventname and data-mbr-eventvalue.

  • Example 1
  • Example 2

When the link is clicked create an ActiveCampaign event with the name clicked-image and log the event in the ActiveCampaign contact record with an event value of logo.png.

<img src="logo.png" data-mbr-eventvalue="logo.png" data-mbr-eventname="clicked-image">

When the link is clicked create an ActiveCampaign event with the name clicked-link and log the event in the ActiveCampaign contact record with an event value of download info.pdf.

<a href="https://yoursite.com/info.pdf" data-mbr-eventvalue="download info.pdf" data-mbr-eventname="clicked-link">

onSubmit​

To define/set an onSubmit logging event for a HTML form use the HTML5 data attributes data-mbr-form-eventname and data-mbr-form-eventvalue.

  • Example 1

When the form is submitted create an ActiveCampaign event with the name submitted-form and log the event in the ActiveCampaign contact record with an event value of submitted update form.

<form data-mbr-form-eventvalue="submitted update form" data-mbr-form-eventname="submitted-form">
  . *form elements* .
</form>
Important

The onSubmit event is triggered and logged immediately upon submission of the form irrespective of whether the form submitted successfully i.e. before any form validation takes place.

Shortcode​

An ActiveMember360 shortcode [mbr_log_event] is also available for event tracking when a logged in contact visits a page or post.

The [mbr_log_event] shortcode is detailed in ActiveMember360, Shortcodes, mbr_log_event
